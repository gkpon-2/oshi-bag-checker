# Oshi Bag Checker 要件定義書

最終更新: 2026-06-20
バージョン: 1.0（リファクタリング・新機能対応版）

---

## 1. 文書の目的

推し活現場向けの持ち物チェックリスト生成アプリ「Oshi Bag Checker」が満たすべき要件を定義する。CLI版とStreamlit Webアプリ版の両方を対象とする。

---

## 2. プロジェクト概要

### 2.1 プロダクト名

Oshi Bag Checker（推し活現場もちものチェックリスト）

### 2.2 一言説明

現場タイプ・天気・気温・遠征有無を選ぶだけで、推し活現場に必要な持ち物チェックリストを自動生成するアプリ。

### 2.3 背景・課題

ライブ・舞台・接触イベント・オンラインイベントなど、現場ごとに必要な持ち物は少しずつ変わる。さらに天気・気温・遠征の有無でも変わる。毎回手作業でリストを作るのは面倒で、忘れ物も起きやすい。条件を選ぶだけで持ち物リストが整う仕組みがあれば、準備の負担と忘れ物を減らせる。

### 2.4 学習目的（個人開発）

本アプリはPythonの基礎学習と個人開発練習を兼ねて作成された。条件分岐・リスト操作・ファイル出力・簡単なWebアプリ・GitHub管理・Streamlit Cloud公開の習得が狙い。

---

## 3. 想定ユーザー

| 区分 | 説明 |
|------|------|
| 推し活をするファン | ライブ・舞台・接触・オンラインの現場に参加する人 |
| CLI利用者 | ターミナルで手早くリストを作りたい人 |
| Webアプリ利用者 | ブラウザで操作し、チェックやダウンロードをしたい人 |

---

## 4. スコープ

### 4.1 本リリースに含むもの

- 現場タイプ別の持ち物リスト生成（ライブ／舞台／接触／オンライン）
- 天気別の持ち物追加（晴れ／雨）
- 気温別の持ち物追加（普通／暑い／寒い）
- 遠征用の持ち物追加
- 重複した持ち物の自動削除
- チェックリストの画面表示・画面上でのチェック操作
- txtファイルでのダウンロード（Web版）・保存（CLI版）
- **自由追加機能（ユーザーが任意の持ち物を追加でき、txtにも反映）** ← 新規
- **チェック操作後もリストが保持される（初期画面に戻らない）** ← 不具合修正

### 4.2 本リリースに含まないもの（今後の予定）

- 作成したチェックリストの編集（項目の削除・並べ替え）
- 現場名・日付の入力欄
- グッズ購入予定リスト
- 交通・宿泊メモ欄
- 忘れ物防止の重要度表示
- スマホ表示向けのUI最適化

## 5. 機能要件

| ID | 機能 | 内容 |
|------|------|------|
| FR-01 | 現場タイプ選択 | live / stage / meet / online から1つ選ぶ |
| FR-02 | 天気選択 | sunny / rainy から1つ選ぶ |
| FR-03 | 気温選択 | normal / hot / cold から1つ選ぶ |
| FR-04 | 遠征有無選択 | 遠征あり／なしを選ぶ |
| FR-05 | チェックリスト生成 | 選択条件に応じた持ち物を集約し、重複を除く |
| FR-06 | 基本持ち物の付与 | 現場に関わらず必要な基本持ち物を必ず含める |
| FR-07 | 条件のおさらい表示 | 選んだ条件を画面・txtに表示する |
| FR-08 | メモ項目の付与 | 「開演時間確認」等の確認メモを末尾に付ける |
| FR-09 | 画面表示・チェック操作 | 持ち物・メモをチェックボックスで表示 |
| FR-10 | txtダウンロード（Web） | チェックリストをtxtで保存できる |
| FR-11 | txt保存（CLI） | outputフォルダへ日時付きで保存できる |
| FR-12 | 自由追加（新規） | 任意の持ち物を入力して追加でき、生成リスト・txtの両方に反映される |
| FR-13 | 作り直し（新規） | 条件を変えて作り直すためのリセットができる（Web） |

---

## 6. 非機能要件

| ID | 区分 | 内容 |
|------|------|------|
| NFR-01 | 保守性 | ロジックを責務ごとにモジュール分割し、持ち物データの追加が容易であること |
| NFR-02 | 可読性 | 各処理に日本語コメントを付け、学習者が読んで理解できること |
| NFR-03 | 信頼性 | Web版でチェック操作をしても作成済みリストが消えないこと |
| NFR-04 | 移植性 | CLIとWebでロジック（oshi_bagパッケージ）を共用すること |
| NFR-05 | 公開性 | Streamlit Cloud上で公開・更新できること |

---

## 7. 制約条件

- 使用言語はPython（学習目的のため基本的な文法中心）
- Webフレームワークは Streamlit
- 外部DBは使用せず、状態はセッション内・ファイル出力で完結
- ダウンロード形式は txt（プレーンテキスト）

---

## 8. 不具合修正要件

| ID | 事象 | 原因 | 対応 |
|------|------|------|------|
| BUG-01 | チェックボックスを押すと初期画面に戻り、チェックリストが消える | 表示判定をフォーム送信フラグ（押した瞬間だけTrue）のみで行っていたため、チェック操作による再実行でFalseに戻り初期表示へ分岐していた | 生成結果を `st.session_state` に保存し、保存済みデータの有無で表示判定するように変更 |

| 用語 | 説明 |
|------|------|
| 現場 | ライブ・舞台・接触などイベントが行われる場 |
