## 2. システム構成

### 2.1 全体像

Oshi Bag Checker は、共通のロジック（`oshi_bag` パッケージ）を、CLIとStreamlit Webの2つの入口（エントリポイント）から呼び出す構成とする。

```
            ┌──────────────────┐
            │  oshi_bag（共通ロジック） │
            │  data / checklist /     │
            │  output / custom_items  │
            │  / cli                  │
            └──────────────────┘
              ▲                ▲
              │                │
   ┌──────────┘                └──────────┐
   │                                       │
[main.py]                          [streamlit_app.py]
 CLI入口                              Web入口（Streamlit）
```

### 2.2 技術スタック

| 区分 | 採用 |
|------|------|
| 言語 | Python 3 |
| Webフレームワーク | Streamlit |
| 標準ライブラリ | datetime（日付）, pathlib（ファイル保存） |
| バージョン管理 | Git / GitHub |
| 公開 | Streamlit Community Cloud |

### 2.3 ファイル構成

```
oshi-bag-checker/
├── main.py              # CLIエントリ（起動のみ）
├── streamlit_app.py     # Webエントリ（画面まわり）
├── requirements.txt
├── README.md
├── .gitignore
├── output/              # CLI版のtxt保存先（git管理外）
└── oshi_bag/            # ロジックパッケージ
    ├── __init__.py
    ├── data.py          # 持ち物マスタ・ラベル定義
    ├── checklist.py     # チェックリスト生成・重複削除
    ├── output.py        # txt整形・ファイル保存
    ├── custom_items.py  # 自由追加アイテムの整理（新機能）
    └── cli.py           # CLI対話処理
```

## 3. モジュール分割方針

「データ」「ロジック」「入出力」「画面」を分離し、変更の影響範囲を局所化する。

| モジュール | 責務 | 変更が必要になる場面 |
|------------|------|---------------------|
| data.py | 持ち物マスタ・日本語ラベルの定義 | 持ち物や選択肢を増やすとき |
| checklist.py | 条件から持ち物リストを組み立てる | 組み立てルールを変えるとき |
| custom_items.py | 自由追加テキストをリストに整形 | 入力の受け取り方を変えるとき |
| output.py | txt整形・保存 | 出力フォーマットを変えるとき |
| cli.py | CLIの対話・表示 | ターミナルの操作感を変えるとき |
| main.py | CLI起動 | （基本変更なし） |
| streamlit_app.py | Web画面・状態管理 | Web UIを変えるとき |

ロジック（oshi_bag）はUIに依存しないため、CLIとWebで共用できる。

## 4. データ設計

### 4.1 持ち物マスタ（data.py）

| 定義名 | 型 | 内容 |
|--------|----|------|
| BASE_ITEMS | list | 基本の持ち物（必ず付与） |
| EVENT_ITEMS | dict | 現場タイプ別の持ち物（キー: live/stage/meet/online） |
| WEATHER_ITEMS | dict | 天気別の持ち物（キー: sunny/rainy） |
| TEMPERATURE_ITEMS | dict | 気温別の持ち物（キー: normal/hot/cold） |
| TRAVEL_ITEMS | list | 遠征用の持ち物 |
| MEMO_ITEMS | list | 末尾に付ける確認メモ項目 |

### 4.2 表示ラベル（data.py）

| 定義名 | 内容 |
|--------|------|
| EVENT_LABELS | 現場タイプのキー→日本語 |
| WEATHER_LABELS | 天気のキー→日本語 |
| TEMPERATURE_LABELS | 気温のキー→日本語 |

キー（内部値）と表示（日本語）を分けることで、内部処理を英字キーで安定させつつ、画面表示を日本語にできる。

### 4.3 状態管理（Web版）

Streamlitは操作のたびにスクリプトを再実行するため、再実行をまたいで保持したいデータは `st.session_state` に保存する。

| session_stateキー | 内容 |
|-------------------|------|
| checklist_result | 生成済みの条件・チェックリスト・出力テキストをまとめた辞書 |
| item_<持ち物名> | 各持ち物チェックボックスの状態 |
| memo_<メモ名> | 各メモチェックボックスの状態 |

## 5. 主要処理方式

### 5.1 チェックリスト生成

1. 基本持ち物を入れる
2. 現場タイプ別を足す
3. 天気別を足す
4. 気温別を足す
5. 遠征ありなら遠征用を足す
6. 自由追加分があれば足す
7. 重複を除去して返す（先勝ちで順序維持）

### 5.2 自由追加の整理

入力テキストを改行・カンマ（半角／全角）で分割し、前後空白除去・空文字除去・重複除去を行ってリスト化する。生成リストにもtxt出力にも同じ結果が反映される。

### 5.3 txt出力

条件のおさらい→持ち物（□付き）→メモの順でテキスト化する。Web版はダウンロードボタンで配布、CLI版は output フォルダへ日時付きで保存する。

### 5.4 Web版の表示制御（不具合対策）

生成結果を `st.session_state["checklist_result"]` に保存し、表示は「送信ボタンが押されたか」ではなく「保存済み結果が存在するか」で判定する。これにより、チェックボックス操作による再実行でも表示が保持される。

## 6. 画面構成（Web版）

| 区画 | 内容 |
|------|------|
| ヘッダー | タイトル・説明文 |

| 結果表示 | 条件おさらい／持ち物チェック／メモチェック／txtダウンロード／作り直しボタン |

## 7. 公開・運用方式

- GitHubリポジトリをStreamlit Community Cloudに接続し、指定ブランチのpushで自動デプロイ
- `requirements.txt` に streamlit を記載（依存追加時はここに追記）
- `output/` はローカルのCLI保存用で、git管理外（.gitignore）
